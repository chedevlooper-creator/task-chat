# Plan 01-01: Formalize Design Tokens

## Objective
Audit and consolidate the existing Tailwind design tokens and CSS utilities into a documented, consistent design system. Ensure all spacing, typography, color, and animation tokens are properly declared in tailwind.config.js and src/styles.css.

## Why This Plan
The project already has a mature obsidian glass design system but it's partially documented. The Tailwind config has colors, fonts, border-radius, shadows, and animations. The styles.css has glass variants, utility classes, accessibility rules, and safe-area padding. This plan audits what exists, fills gaps, and ensures consistency with the UI-SPEC.md contract.
