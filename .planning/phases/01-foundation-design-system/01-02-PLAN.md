# Plan 01-02: Audit and Align Component Library

## Objective
Verify that all existing React components follow the obsidian glass design system documented in UI-SPEC.md. Audit card, button, badge, and chip components for consistency with the design contract. Identify and fix any deviations.

## Why This Plan
The project has existing components (Header, Badge, TaskRow, Avatar, FilterBar, DaySection, TaskDetailModal, ChatPanel, TeamPanel, RemindersPanel) that use the glass system. This plan ensures every component uses the documented design tokens, border radius values, spacing, and interaction patterns from UI-SPEC.md rather than arbitrary inline styles or inconsistent Tailwind classes.

## Tasks

### Task 1: Audit Card Component Usage

<read_first>
- src/styles.css (lines 172-203 for .card, .card-2, .glass, .glass-strong, .glass-soft definitions)
- src/App.tsx (DaySkeleton component — uses .glass class)
- src/components/Header.tsx (MetricCard, PulseStat, DayLoad components — uses .glass-soft)
- src/components/TaskRow.tsx (uses .glass-soft)
- src/components/DaySection.tsx
- src/components/TaskDetailModal.tsx
- src/components/ChatPanel.tsx
- src/components/TeamPanel.tsx
- src/components/RemindersPanel.tsx
- .planning/phases/01-foundation-design-system/01-UI-SPEC.md (Component Inventory > Cards section)
</read_first>

<action>
Audit all components that use card-like styling. For each component:

1. Verify the correct glass variant is used per UI-SPEC.md:
   - `.glass` for primary content cards (backdrop-blur 26px)
   - `.glass-strong` for modals and nav bars (backdrop-blur 32px)
   - `.glass-soft` for metric cards and inline elements (backdrop-blur 18px)

2. Verify border radius matches the UI-SPEC.md Border Radius Scale:
   - Primary cards should use `rounded-[26px]` (xl) or `.card` class
   - Secondary cards should use `rounded-[22px]` (lg) or `.card-2` class
   - Metric cards use `rounded-[24px]` (between lg and xl — acceptable)

3. Check that no component uses arbitrary inline styles for shadows, borders, or backgrounds that should use the CSS utility classes instead.

4. Replace any inconsistent usage with the correct utility class.
</action>

<acceptance_criteria>
- grep -rn "glass" src/components/ shows consistent usage of .glass, .glass-strong, .glass-soft classes
- No component uses inline box-shadow for card effects (should use CSS classes)
- grep -rn "rounded-\[" src/components/ shows border-radius values that match UI-SPEC.md scale (8, 12, 14, 18, 22, 24, 26, 28, 32, 34)
- All card containers use one of the three glass utility classes
- At least 3 components use .glass-soft (MetricCard, PulseStat, TaskRow confirmed)
</acceptance_criteria>
